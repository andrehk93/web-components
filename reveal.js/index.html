Hello world!

Reusable

  • Like any other HTML element
  • Can be exposed by script, NPM, etc.
  • Buttons, headers, forms, menus ++
Can be inserted into the DOM as many times as needed
If it works in one application - it works in "all"
Perfect for adding new modern functionality into legacy applications
Can be exposed by script for "live" distributed updates
Can also be exposed via package managers
Can be anything from small buttons to large forms, menus, etc.

Fully encapsulated

  • CSS isolation!
  • Agnostic to framework
  • DevTools
Offers complete CSS isolation, DOM isolation, Debug capabilities, etc.
Makes it thus agnostic to framework, but bear in mind some are more supported than others
Ideal for settings where common dependencies creates issues with versioning etc. (e.g. design systems)

How it works

3 main technologies combined

  • Custom elements
  • Shadow DOM
  • HTML Templates
Custom elements - a registry of additional HTML elements to render
Must be hyphenated, e.g. itera-header
Shadow DOM - Empowers Web Components with encapsulation, and offers a separated DOM for the WC
Enables "private" execution of scripts and styles within the Shadow DOM and not the original DOM
HTML Templates - Supports HTML template and slot elements, but I won't cover those here
